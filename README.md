# Sylntra Website

> **Better systems. Not more software.**

## Overview

The Sylntra Website is the official digital presence of Sylntra, a Business Systems Studio focused on helping businesses improve efficiency through modern websites, intelligent automation, and scalable digital systems.

This repository contains the complete frontend source code, project documentation, engineering standards, and development workflow for the Sylntra website.

The project is intentionally built using an incremental engineering approach where every improvement is planned, reviewed, committed, and documented.

---

# Project Goals

The website exists to:

* Position Sylntra as a premium Business Systems Studio.
* Build trust with potential clients.
* Demonstrate systems thinking rather than technical complexity.
* Generate qualified consultation requests.
* Create reusable frontend assets for future Sylntra projects.

---

# Core Philosophy

Sylntra does not sell websites.

Sylntra builds business systems.

Every page should communicate business value before technology.

Technology is the implementation.

Business transformation is the product.

# Git Workflow

Every meaningful improvement should be committed separately.

Examples:

* Improve Hero Layout
* Add Founder Portrait
* Refine Navigation
* Optimize Mobile Experience

Avoid large mixed commits.

# Working with Claude Code

Claude Code should:

1. Read `CLAUDE.md` before making changes.
2. Work on one milestone at a time.
3. Reuse existing components whenever possible.
4. Explain the implementation plan before coding.
5. Preserve the design system.
6. Keep commits small and meaningful.
